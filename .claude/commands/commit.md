# Git 커밋 자동화

현재 변경사항을 분석하고 conventional commit 메시지로 커밋하세요.

1. `git status`로 변경사항 확인
2. `git diff`로 변경내용 검토
3. 적절한 커밋 타입 결정 (feat, fix, docs 등)
4. `git add -A`로 스테이징
5. conventional commit 형식으로 커밋

## conventional commit message

```
🎉 Init: 프로젝트 초기 세팅, 첫 커밋
✨ Feat: 새로운 기능 추가
➕ Chore: 새로운 라이브러리/패키지 설치
🍱 Asset: 이미지/리소스 파일 추가
♻️ Refactor: 코드 리팩터링 (기능 변화 없음)
🐛 Bug: 버그 수정
🚑 HOTFIX: 바로 변경이 필요한 버그 수정
🔥 Remove: 불필요한 코드/파일 제거
📝 Docs: 문서 수정
✏️ Typo: 오타 수정
---
DEPLOY
✅ Test: 테스트 코드 추가, 업데이트, 통과
🚀 Deploy: 실제 서비스 배포 관련 커밋
🔧 CI: GitHub Actions, Jenkins 등 CI 설정 변경
📦 Build: 빌드 시스템, 환경설정 (Vite, Webpack, Babel 등) 수정
🔒 Security: 보안 관련 패치 (패키지 업데이트, 취약점 해결 등)
---
BE
🗃️ DB: DB 스키마, 마이그레이션, 쿼리 변경
🔐 Auth: 인증/인가 로직 추가 및 수정
🌐 API: API 엔드포인트 추가/변경
🛠️ Server: 서버 설정, 미들웨어, 인프라 코드 변경
---
FE
🎨 UI: UI/스타일 변경 (CSS, 디자인 관련)
🚸 UX: UX 향상
⚡ Perf: 성능 개선 (렌더링 최적화, 코드 스플리팅, SEO 등)
♿️ A11y: 접근성 개선
```
